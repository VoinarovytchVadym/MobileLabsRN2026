# Лабораторна робота №1

## Опис проєкту

Цей проєкт створений у середовищі **Expo** для розробки простого мобільного додатку **React Native**. Додаток включає базові екрани, такі як "Головна", "Галерея", "Профіль" та екран реєстрації, що дозволяє користувачу вводити свої дані.

Проєкт реалізує використання компонентів **React Native** для створення мобільних додатків, а також базову навігацію між екранами за допомогою **`@react-navigation/native`**.

## Інструкція із запуску

### Встановлення середовища

1. **Встановіть Expo CLI:**
   ```bash
   npm install -g expo-cli
2. **Клонування репозиторію:**
   ```bash
   git clone https://github.com/ваш_профіль/MobileLabsRN2026.git
   cd MobileLabsRN2026/lab1
3. **Встановлення залежностей:**
   ```bash
   npm install
4. **Запуск додатку:**
   ```bash
   npm start
5. **Відкрийте Expo Go на вашому мобільному пристрої і відскануйте QR-код, що відображається в консолі, або використовуйте емулятор для запуску додатку.**

### Пояснення:
1. **Інструкція із запуску**:
   - Встановлення **Expo CLI**, клонується репозиторій, встановлюються залежності та запускається проект.
2. **Запуск на фізичному пристрої**:
   - Завантаження **Expo Go** на реальний пристрій і використання QR-коду для запуску додатку.
3. **Структура проєкту**:
   - Головна:
     * Відображає список новин з їх заголовками, датою та коротким текстом. Це перший екран, що бачить користувач після запуску додатку.
   - Галерея:
     * Покажує зображення в вигляді плитки. Користувач може переглядати зображення, що містяться в галереї.
   - Профіль:
     * Екран реєстрації, де користувач може ввести свою електронну пошту, пароль, ім'я та прізвище для створення облікового запису.
4. **Скріншоти**:
   
5. **Особливості тестування**:
   - Expo Go (реальний пристрій):
     * Призначення: Простий спосіб тестування на реальному пристрої.
     * Особливості: Легкий запуск, без необхідності в складному налаштуванні. Підтримка базових функцій.
     * Тестування: Ідеальний для швидкого тестування інтерфейсу та основних функцій на реальних пристроях.
   - Емулятор Android (Android Studio):
     * Призначення: Для тестування на віртуальному пристрої.
     * Особливості: Тестування без фізичного пристрою. Можна перевіряти різні версії Android.
     * Тестування: Може бути повільнішим, але зручний для тестування додатка на різних версіях Android.
